# Техническа спецификация за разработка на нов уеб сайт на Школа А&Б

## 1. Общ преглед и цели

### 1.1 Предистория
Школа А&Б е утвърден образователен център, специализиран в обучението по математика и информатика в град Шумен, с над 25-годишна история. Школата е известна с високите постижения на своите възпитаници на национални и международни състезания по програмиране.

### 1.2 Цели на новия уебсайт
- Създаване на модерен, отзивчив и достъпен уебсайт, който отразява високото ниво на обучение в школата
- Осигуряване на лесен достъп до информация за програмите, методиката и постиженията
- Улесняване на комуникацията между школата, ученици и родители
- Осигуряване на лесна поддръжка и актуализация на съдържанието
- Оптимизация за търсачки (SEO) за по-добро присъствие онлайн
- Двуезична поддръжка (български и английски)

## 2. Технологичен стек

### 2.1 Фронтенд
- **Основна технология**: Next.js 14 (React framework)
- **Език за програмиране**: TypeScript
- **Стилизация**: Tailwind CSS + Shadcn UI компоненти
- **Управление на форми**: React Hook Form + Zod валидация
- **Анимации**: Framer Motion
- **Икони**: Lucide React
- **Карти (ако са необходими)**: Leaflet или Google Maps API

### 2.2 Бекенд
- **Сървърна рамка**: Next.js API Routes (за по-малки заявки) и Node.js с Express (за по-сложни операции)
- **База данни**: MongoDB (за динамично съдържание като новини, постижения, събития)
- **CMS**: Самостоятелно разработена административна панел или интеграция с Contentful/Sanity
- **Системa за управление на файлове**: Uploadcare или AWS S3
- **Управление на сесии/автентикация**: NextAuth.js

### 2.3 Инфраструктура и DevOps
- **Хостинг**: Vercel (основна опция), алтернативно AWS или DigitalOcean
- **CI/CD**: GitHub Actions
- **Мониторинг**: Sentry за проследяване на грешки, Google Analytics за потребителско поведение
- **Контрол на версиите**: Git/GitHub
- **Контейнеризация**: Docker (опционално, за локално разработване и потенциално деплойване)

## 3. Архитектура на уебсайта

### 3.1 Архитектурен подход
Сайтът ще използва хибридна архитектура, комбинирайки:
- **Статично генерирани страници (SSG)** за съдържание, което рядко се променя (история, за нас, методика)
- **Сървърни компоненти (RSC)** за динамично съдържание (новини, постижения, събития)
- **Клиентски компоненти** само когато е необходима интерактивност (форми за контакт, анимации)

### 3.2 Структура на данните
1. **Новини**:
   ```typescript
   interface News {
     id: string;
     title: string;
     content: string;
     date: Date;
     imageUrl?: string;
     tags: string[];
     isHighlighted: boolean;
   }
   ```

2. **Постижения**:
   ```typescript
   interface Achievement {
     id: string;
     studentName: string;
     competition: string;
     group: 'A' | 'B' | 'C' | 'D' | 'E';
     rank: number;
     year: number;
     isInternational: boolean;
   }
   ```

3. **Преподаватели**:
   ```typescript
   interface TeamMember {
     id: string;
     name: string;
     position: string;
     education: string;
     imageUrl?: string;
     bio: string;
   }
   ```

4. **Курсове**:
   ```typescript
   interface Course {
     id: string;
     title: string;
     description: string;
     ageGroup: string;
     duration: string;
     schedule: string[];
     isActive: boolean;
     syllabus: string[];
   }
   ```

### 3.3 API Endpoints
1. GET `/api/news` - Извличане на всички новини
2. GET `/api/news/:id` - Извличане на конкретна новина
3. GET `/api/achievements` - Извличане на постижения с филтриране
4. GET `/api/team` - Извличане на информация за екипа
5. GET `/api/courses` - Извличане на информация за курсовете
6. POST `/api/contact` - Изпращане на форма за контакт

## 4. Структура на сайта и дизайн

### 4.1 Карта на сайта
- **Начало** (`/`) - Уводна страница с акценти върху последните постижения и новини
- **За нас** (`/about`)
  - История (`/about/history`)
  - Екип (`/about/team`)
- **Обучение** (`/education`)
  - Методика (`/education/methodology`)
  - Възрастови групи (`/education/age-groups`)
  - Учебен процес (`/education/process`)
- **Постижения** (`/achievements`) - С филтри по години, състезания и групи
- **Новини и събития** (`/news`)
- **Галерия** (`/gallery`)
- **Контакти** (`/contacts`)
- **Админ панел** (`/admin`) - защитен с парола раздел за управление на съдържанието

### 4.2 Дизайн принципи
- **Цветова схема**: Основана на логото на школата, с акцент върху сини и зелени тонове (професионални и образователни)
- **Типография**: 
  - Заглавия: Montserrat или Nunito
  - Тяло: Inter или Open Sans
  - Моноширинен шрифт (за код примери): JetBrains Mono или Fira Code
- **Визуален стил**: Чист, минималистичен дизайн с фокус върху съдържанието
- **Анимации**: Умерени, ненатрапчиви преходи за подобряване на потребителското изживяване

### 4.3 Компоненти и макети на страниците
Основният макет ще включва:
- Фиксирано навигационно меню с падащи подменюта (responsive)
- Hero секция на началната страница с въртящи се акценти
- Секция за последни новини/събития
- Секция за изтъкнати постижения
- Футър с контактна информация и връзки към социални мрежи

## 5. Адаптивен дизайн

### 5.1 Breakpoints
- **Mobile**: 320px - 639px
- **Tablet**: 640px - 1023px
- **Desktop**: 1024px - 1279px
- **Large Desktop**: 1280px+

### 5.2 Подход за адаптивност
- Mobile-first approach
- Използване на Flexbox и CSS Grid за гъвкаво оформление
- Tailwind CSS responsive класове
- Оптимизирани изображения с различни размери за различни устройства (използвайки Next.js Image компонент)

## 6. Производителност и оптимизация

### 6.1 Performance оптимизации
- Code-splitting и lazy loading на компоненти
- Оптимизация на изображения (WebP формат, автоматични размери)
- Използване на ISR (Incremental Static Regeneration) за полу-динамично съдържание
- Минимизиране на JavaScript bundle size чрез tree-shaking
- Използване на кеширане за API заявки

### 6.2 SEO оптимизации
- Метаданни за всяка страница (title, description)
- Структурирани данни (JSON-LD)
- Семантичен HTML
- Sitemap.xml и robots.txt
- Оптимизирани URL-и (чисти и дескриптивни)
- Поддръжка на OpenGraph тагове за социални мрежи

## 7. Интернационализация

### 7.1 Многоезичност
- Начална поддръжка на български и английски
- Използване на next-intl за превод
- Структура на локализация:
  ```
  /locales
    /bg
      common.json
      home.json
      about.json
      ...
    /en
      common.json
      home.json
      about.json
      ...
  ```

### 7.2 URL структура с локализация
- `/bg/...` - български URLs
- `/en/...` - английски URLs
- Автоматично пренасочване към предпочитания език на потребителя

## 8. Admin Panel/CMS

### 8.1 Функционалности на административния панел
- Управление на новини (създаване, редактиране, изтриване)
- Управление на постижения (добавяне, редактиране, архивиране)
- Управление на информация за курсове
- Качване и организиране на изображения за галерията
- Просмотр и експорт на съобщения от формата за контакт

### 8.2 Автентикация и сигурност
- Защитен достъп с роли (администратор, редактор)
- Two-factor authentication
- Audit logs за проследяване на промените
- CSRF защита
- Rate limiting за предотвратяване на brute force атаки

## 9. Интеграции

### 9.1 Електронна поща
- Интеграция с SMTP сървър или SendGrid за изпращане на имейли от формата за контакт
- Възможност за бюлетин/newsletter (опционално)

### 9.2 Социални мрежи
- Бутони за споделяне на съдържание
- Вграждане на социални канали (Facebook feed, Instagram gallery)

### 9.3 Карти
- Интеграция с Google Maps или OpenStreetMap за страницата с контакти

## 10. Тестване

### 10.1 Тестови стратегии
- **Unit тестове**: Jest + React Testing Library за компоненти
- **E2E тестове**: Cypress за критични пътища на потребителя
- **API тестове**: Supertest за API endpoints
- **Визуални тестове**: Storybook + Chromatic (опционално)
- **Accessibility тестове**: axe-core или Lighthouse

### 10.2 Браузър съвместимост
- Chrome, Firefox, Safari, Edge - последни 2 версии
- IE не се поддържа

   - Accessibility тестване

4. **Пускане на живо и финализиране** (1 седмица)

## 13. Изисквания за хостинг и домейн

### 13.1 Хостинг изисквания
- Node.js >= 18.x
- Минимум 1GB RAM
- Поддръжка на HTTPS
- MongoDB или Atlas връзка

### 13.2 Домейн и DNS
- Използване на съществуващия домейн (ab-bg.com)
- Конфигуриране на DNS записи:
  - A запис за основния домейн
  - CNAME за www поддомейн
  - MX записи за e-mail услуги
  - TXT записи за SPF и DKIM (ако се използва e-mail)

## 14. Сигурност

### 14.1 Мерки за сигурност
- HTTPS с автоматични подновяващи се SSL сертификати
- Защита от основните уеб уязвимости (XSS, CSRF, SQL injection)
- Content-Security-Policy (CSP) хедъри
- Rate limiting за API заявки
- Защита на административния панел с най-малко привилегии
- Редовни актуализации на пакети за поправяне на уязвимости

### 14.2 Съответствие с регулации
- GDPR съвместимост (политика за поверителност, съгласие за бисквитки)
- Достъпност по WCAG 2.1 (минимум ниво AA)
